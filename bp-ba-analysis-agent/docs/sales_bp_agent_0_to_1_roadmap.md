# Sales BP 分析 Agent 0 到 1排期计划

## 1. 计划口径

本计划按 2026-05-22 的项目状态估算，默认从 2026-05-25 开始执行。当前已有 Python 单服务原型、三 Agent 工作流、任务管理页面、能力目录、Ontology 语义维护、CSV 真实数据 Demo 和 HTML 报告导出能力，因此排期不是从空仓库开始，而是从现有 POC 收敛到可试点产品。

团队假设：

- BA / 产品 / 业务协调：1 人。
- 开发：最多 1 人。
- 使用 AI coding 提升开发、测试和文档产出效率。
- MVP 阶段以 CSV + 语义配置为数据边界，不强依赖企业数仓。

版本目标：

- MVP 试点版：8-10 周，2026-05-25 到 2026-07-31。
- 生产级系统：5-6 个月，2026-05-25 到 2026-11-30。

MVP 目标不是替代所有 Sales BP BA 工作，而是让 BA 能基于真实 CSV 与可维护语义配置完成完整分析闭环：创建任务、分析思路、数据洞察、报告生成、能力沉淀。

## 2. MVP 试点版排期

### 第 1 周：产品定义与架构冻结

时间：2026-05-25 到 2026-05-29

目标：把当前原型收敛成 MVP 需求基线。

交付：

- 明确 3 个首批业务场景：客流下降分析、订单/转化下降分析、渠道或经销商表现分析。
- 固定 4 个页面节点：创建分析任务、分析思路设计、数据分析与洞察、报告生成。
- 固定 3 个 Agent：分析思路 Agent、数据分析 Agent、报告生成 Agent。
- 输出 MVP PRD / 验收清单。
- 确认 CSV 数据样例、字段字典、可用指标、不可用数据边界。

验收：

- BA 能说明 MVP 做什么、不做什么。
- 首批 3 个场景都有样例问题、所需指标、维度和数据可用性说明。
- 开发可以按冻结范围推进，不需要继续猜产品边界。

### 第 2-3 周：任务管理与分析输入闭环

时间：2026-06-01 到 2026-06-12

目标：让 BA 能稳定创建、保存、打开、复制、删除分析任务。

交付：

- 任务管理页产品化：搜索、状态筛选、任务进度、草稿/进行中/已完成状态。
- 创建分析任务页：业务问题、任务名称、分析目标、数据源选择、时间范围、对比基期、分析维度多选。
- BA 确认后才启动分析思路 Agent。
- 本地 JSON 持久化稳定化：任务输入、当前节点、BA 确认记录、手工编辑内容。
- 任务输入变化后，提示重新生成分析思路。

验收：

- 创建草稿不会触发 Agent。
- 确认任务输入后才进入分析思路 Agent。
- 服务重启后任务列表和任务详情可恢复。
- 修改已确认任务输入时能提示影响后续节点。

### 第 4-5 周：分析思路 Agent 产品化

时间：2026-06-15 到 2026-06-26

目标：让 AI 输出像 BA 的分析思路，而不是固定模板。

交付：

- 分析思路拓展页：AI 对业务问题的理解、分析目标、归因路径、归因框架、数据可用性提示。
- 成熟分析框架页：可选择框架、指标模板、维度模板、数据要求。
- 动态假设生成：按业务问题主题变化，并支持 BA 新增、编辑、删除、改状态。
- 能力目录可维护：问题类型、分析方法、Playbook、指标模板。
- Ontology 语义维护可维护：业务对象、业务事件、对象关系、指标口径。

验收：

- 换不同业务问题后，分析思路和假设明显变化。
- BA 修改假设后，后续数据分析使用修改后的假设。
- 能力目录和 Ontology 能新增、编辑、删除、保存、重载。
- 分析思路中明确标识哪些数据可用、哪些需要补充。

### 第 6-7 周：数据分析与洞察闭环

时间：2026-06-29 到 2026-07-10

目标：基于 CSV 生成真实可查看的数据分析结果。

交付：

- 数据源能力：使用当前 Sales 宽表 CSV，支持字段识别和基础映射，显示字段可用性和缺失项。
- 数据分析页 4 个 Tab：指标概览、维度指标表、假设验证、计算逻辑 SQL。
- 指标概览：分组指标卡片、风险识别点、图表预览、数据质量报告。
- 假设验证：跟随分析主题变化，每条假设绑定证据指标，并标记已验证、待验证、已否定。
- SQL 逻辑：可审计、可复制，作为技术明细而不是默认主视图。

验收：

- 数据分析默认进入指标概览。
- 风险识别点必须有指标证据支撑。
- 假设验证不能复用固定文案，必须和分析主题一致。
- SQL Tab 能解释核心指标计算逻辑。

### 第 8 周：报告生成与导出

时间：2026-07-13 到 2026-07-17

目标：生成 BA 可编辑、业务能阅读的报告。

交付：

- 报告生成页：执行摘要、回顾背景、数据概览、关键发现、原因分析、建议与结论。
- 报告编辑区：支持保存编辑内容，支持重新打开恢复。
- 图表预览：核心指标图、渠道/客户/漏斗图。
- HTML 真实导出。
- PPT/PDF/Word 保留入口，明确标注后续支持。

验收：

- 报告内容可编辑、保存、重新打开。
- HTML 报告可导出并访问。
- 报告中结论和图表能对应到数据洞察结果。
- 未真实支持的导出格式不能表现成可用功能。

### 第 9 周：试点验收与修正

时间：2026-07-20 到 2026-07-24

目标：用真实业务问题跑通试点。

交付：

- 至少跑通 3 个案例：客流下降、订单下降、渠道/经销商异常。
- BA 逐步确认：创建任务输入、分析思路、数据洞察、报告是否可直接修改使用。
- 修复关键体验问题。
- 输出试点复盘：节省了哪些 BA 工作、哪些能力仍需人工、下一阶段需要接哪些数据。

验收：

- 3 个案例都能完整跑完 4 个页面节点。
- 每个案例都能产出可读报告。
- 关键问题被记录到下一阶段 Backlog。

### 第 10 周：MVP 封版

时间：2026-07-27 到 2026-07-31

目标：形成可持续演示和小范围试用版本。

交付：

- MVP 版本冻结。
- 操作手册。
- Demo 数据和案例包。
- 已知限制清单。
- 下一阶段 Backlog。
- 内部演示材料。

验收：

- 非开发人员能按操作手册跑通 Demo。
- 已知限制清楚，不把未完成能力包装成已完成能力。
- MVP 可以进入小范围试用或生产化评审。

## 3. 生产级系统排期

### 阶段 1：MVP 试点

时间：2026-05-25 到 2026-07-31

内容：沿用 MVP 计划，作为生产化基础。

### 阶段 2：工程架构生产化

时间：2026-08-03 到 2026-08-28

目标：从单文件原型升级为可维护系统。

交付：

- 前后端工程拆分。
- 后端服务模块化：Task Service、Agent Orchestration Service、Semantic/Ontology Service、Data Profiling Service、Report Service。
- 数据持久化从 JSON 升级为数据库。
- 引入 dev、test、prod 环境配置。
- 增加日志、异常处理、任务执行状态追踪。

### 阶段 3：数据接入与语义层生产化

时间：2026-08-31 到 2026-09-25

目标：从 CSV 试点扩展到企业数据体系。

交付：

- CSV 上传能力生产化：文件校验、字段识别、字段映射、数据预览。
- 语义层维护后台：业务对象、业务事件、指标口径、物理字段映射。
- 预留数仓连接接口。
- 数据质量检测：缺失率、唯一性、时间范围、异常值、敏感字段识别。
- 数据权限方案草案。

### 阶段 4：Agent 能力增强

时间：2026-09-28 到 2026-10-23

目标：提升分析质量和复用能力。

交付：

- 分析思路 Agent：更强的问题分类、动态方法组合、数据可用性约束下的分析计划。
- 数据分析 Agent：多表/多 CSV 分析、指标计算计划、假设验证证据链、风险识别。
- 报告生成 Agent：按不同受众生成不同报告结构、结论强度标注、数据限制说明。
- 能力沉淀：将完成任务沉淀为 Playbook，支持复用历史分析框架。

### 阶段 5：权限、审计、协作与运维

时间：2026-10-26 到 2026-11-20

目标：达到企业内部可控试运行标准。

交付：

- 用户登录与角色权限。
- BA、业务方、管理员角色区分。
- 操作审计：创建任务、确认分析思路、修改报告、导出结果。
- Agent 输出审计：输入、输出、使用数据、版本。
- 任务协作：评论、确认记录、修改历史。
- 服务监控：接口错误、任务失败、Agent 调用失败、导出失败。

### 阶段 6：生产试运行与上线

时间：2026-11-23 到 2026-11-30

目标：完成生产级试运行。

交付：

- UAT。
- 安全检查。
- 性能检查。
- 用户培训。
- 上线手册。
- 回滚方案。
- 生产试运行报告。

## 4. 接口边界

MVP 阶段需要稳定这些接口边界：

- 任务接口：创建草稿任务、更新任务输入、确认任务输入、打开任务、复制任务、删除任务。
- Agent 工作流接口：启动分析思路 Agent、保存分析思路修改、确认分析思路、获取数据分析结果、确认数据洞察、保存报告内容、导出 HTML 报告。
- 配置接口：获取/维护能力目录、获取/维护 Ontology、获取/维护分析框架。
- 数据接口：CSV 数据资产预览、字段识别、语义映射、指标计算结果、数据质量结果。

生产级阶段新增：

- 用户与权限接口。
- 审计日志接口。
- 数据源连接接口。
- 任务运行状态接口。
- Agent 调用记录接口。
- 报告版本管理接口。

## 5. 测试与验收

MVP 验收测试：

- 创建任务后不触发 Agent。
- BA 确认任务输入后，才进入分析思路 Agent。
- 换不同业务问题后，分析思路和假设明显变化。
- BA 修改假设后，数据分析假设验证使用修改后的假设。
- 数据分析默认进入指标概览，不默认展示 SQL。
- 风险识别点有指标证据支撑。
- 报告内容可编辑、保存、重新打开。
- HTML 报告可导出并访问。
- 能力目录和 Ontology 可维护。
- 服务重启后任务状态可恢复。

生产级验收测试：

- 多用户并发创建和打开任务。
- 不同角色权限隔离。
- 敏感字段不可泄露。
- Agent 输出可追溯。
- 数据源异常时有明确失败提示。
- 报告导出失败可重试。
- 关键接口有日志、审计和监控。
- UAT 通过至少 5 个真实 Sales BP 分析案例。

## 6. 风险与依赖

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 单开发资源不足 | 排期延长，测试覆盖不足 | MVP 缩小到 CSV 闭环和 3 个核心场景 |
| 真实数据字段不稳定 | 指标和假设验证失准 | 第 1 周冻结样例字段字典和缺失数据边界 |
| Agent 输出质量不稳定 | BA 不信任结论 | 每一步保留 BA 确认和可编辑机制 |
| 生产级权限/审计资源缺失 | 无法进入企业生产环境 | 生产级阶段明确要求 IT、数据、安全资源参与 |
| 导出格式期望过高 | MVP 交付被 PPT/PDF/Word 拖慢 | MVP 只真实支持 HTML，其他格式标注后续支持 |

## 7. 默认取舍

- MVP 优先产品闭环，不优先架构完美。
- MVP 优先 CSV + 语义配置，不优先企业数仓连接。
- MVP 优先 HTML 报告，不优先 PPT/PDF/Word。
- 生产级系统必须补齐权限、审计、监控、数据库和企业数据接入。
