# Sales BP 分析 Agent 方案说明

---

## 幻灯片 1：方案总览

### 标题
**Sales BP 分析 Agent - 智能分析工作台**

### 副标题
业务问题驱动的自动化分析解决方案

### 核心价值（3个要点）

| 维度 | 传统方式 | Agent赋能方式 |
|------|----------|---------------|
| **问题入口** | 固定报表/指标 | 自然语言业务问题 |
| **分析流程** | 手动拆解、取数、分析 | Agent自动生成方案 + BA确认 |
| **成果沉淀** | 一次性报告 | 可复用Playbook/口径库 |

### 目标定位
- **短期目标**：构建通用分析工作台，赋能BA日常工作
- **长期目标**：逐步替代重复性分析流程

## 幻灯片 2：Agent 架构与工作流

### 标题
**三Agent协作架构**

### 架构图

```
┌─────────────────────────────────────────────────────────────┐
│                    分析工作台（前端）                        │
│  [业务问题输入] → [方案确认] → [洞察审核] → [报告生成]        │
└───────────────────────┬─────────────────────────────────────┘
                        │
        ┌───────────────┼───────────────┐
        ▼               ▼               ▼
┌─────────────┐ ┌───────────────┐ ┌─────────────────┐
│ Agent 1     │ │ Agent 2       │ │ Agent 3         │
│ 分析策略    │ │ 数据分析洞察  │ │ 报告生成        │
├─────────────┤ ├───────────────┤ ├─────────────────┤
│ • 问题分类  │ │ • 取数计划    │ │ • 管理层摘要    │
│ • Playbook  │ │ • SQL生成     │ │ • PPT故事线    │
│ • 假设树    │ │ • 分析执行    │ │ • Excel输出    │
│ • 数据评估  │ │ • 洞察生成    │ │ • 多格式导出    │
└─────────────┘ └───────────────┘ └─────────────────┘
```

### 关键特点
1. **每步需BA确认**：确保分析过程可控、透明
2. **数据可用性显式展示**：区分可用/部分可用/需补充
3. **洞察证据链可追溯**：每条结论绑定数据源和计算逻辑
4. **知识沉淀机制**：分析方法、口径、模板可复用

### 支持的分析类型
- 业绩诊断、转化效率、渠道评估、异常检测、Cohort分析

## 幻灯片 3：当前数据支持与扩展计划（可选补充页）

### 当前支持能力
- ✅ 全链路销售漏斗分析（注册→线索→机会→到店→试驾→订单）
- ✅ 多维度拆解（区域、城市、经销商、品牌、车系）
- ✅ 渠道/活动效果分析、跟进时效分析、战败原因分析

### 待补充数据
- 销售目标、投放成本、市场数据、人员产能

### MVP验收标准
1. 任意业务问题转结构化分析计划
2. 自动判断问题类型，组合分析方法
3. 明确告知数据可用性状态
4. 输出可审计、可编辑、可沉淀的分析过程
